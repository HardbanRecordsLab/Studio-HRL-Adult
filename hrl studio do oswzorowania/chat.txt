To bardzo ważne pytanie przy projektach typu "Content Studio", gdzie pliki media (zdjęcia 4K, filmy, podcasty) zajmują dużo miejsca i generują duży transfer.

### Werdykt: Gdzie trzymać media?
Zdecydowanie lepiej trzymać media w zewnętrznej chmurze (np. Cloudinary), a nie bezpośrednio na Vercel.

Dlaczego?

1. Limity Vercel : Darmowy plan Vercel (Hobby) ma limit 100GB transferu miesięcznie. Jeden film 4K może "zjeść" ten limit bardzo szybko.
2. Optymalizacja : Serwisy takie jak Cloudinary automatycznie kompresują zdjęcia i filmy "w locie" (np. zmieniają format na WebP/Avif), co sprawia, że strona ładuje się błyskawicznie na telefonach.
3. Zarządzanie : Łatwiej zarządzać setkami plików w panelu chmury niż w folderze z kodem.
### Najlepsze darmowe opcje (Free Tier)
1. Cloudinary (Wybrana przez Ciebie) :
   
   - Zalety : Najlepsza na świecie optymalizacja obrazu i wideo.
   - Limit : ok. 25GB transferu/miejsca (rozliczane w kredytach).
   - Status : Już masz tam konto i klucze, więc to nasz wybór nr 1.
2. Supabase Storage :
   
   - Zalety : Bardzo prosta w obsłudze, świetna do plików PDF i surowych wideo.
   - Limit : 1GB miejsca / 5GB transferu.
3. Vercel Blob :
   
   - Zalety : Natywna usługa Vercel, zerowa konfiguracja.
   - Limit : 250MB miejsca (bardzo mało na wideo).
### Co skonfigurowałem?
Skonfigurowałem Twoją aplikację pod pełną integrację z Cloudinary :

1. Whitelist w Next.js : Dodałem domenę Cloudinary do next.config.js , aby komponent next/image mógł bezpiecznie wyświetlać zdjęcia z chmury.
2. Utility Helper : Utworzyłem plik cloudinary.ts , który automatycznie generuje zoptymalizowane linki do Twoich plików (dodaje parametry typu q_auto,f_auto ).
3. Zmienne Środowiskowe : Uzupełniłem plik .env.local o Twoje klucze, aby aplikacja mogła łączyć się z Twoim kontem dfn5qgvpy .
### Jak teraz dodawać media?
Zamiast wrzucać pliki do folderu public/video , wrzuć je do panelu Cloudinary, a w kodzie używaj tylko nazwy pliku.

Przykład użycia w kodzie:

```
import { getCloudinaryUrl } from '@/utils/cloudinary';

// Zdjęcie z automatyczną kompresją i szerokością 800px
const myImageUrl = getCloudinaryUrl('nazwa_pliku_w_chmurze', { width: 800 });
```
### Następny krok na Vercel:
Pamiętaj, aby w panelu Vercel (Settings -> Environment Variables) dodać nową zmienną:

- NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME = dfn5qgvpy
Dzięki temu Twoja strona będzie "lekka" i nie przekroczy limitów transferu Vercel! Czy chciałbyś, abym pokazał Ci, jak przerobić konkretny komponent (np. Hero), aby brał wideo z Cloudinary zamiast z lokalnego folderu?
